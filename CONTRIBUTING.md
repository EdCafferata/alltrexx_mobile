# Contributing

## Branches
- `main` — stabiele productieversie
- `develop` — actieve ontwikkeling
- `feature/*` — nieuwe features

## Commit stijl
```
type: korte beschrijving

Langere uitleg indien nodig.
```
Types: `feat`, `fix`, `refactor`, `docs`, `chore`

## Pull requests
- Altijd via PR naar `develop`
- Minimaal één review voor merge naar `main`

## Code stijl
- Volg de bestaande codeerstijl
- Voeg commentaar toe bij complexe logica
